# 過去問との類似防止ルール

問題生成時に、以下のチェックを必ず実施すること。

## チェック1: トピック名の重複
- `docs/topic-history.md` に記載されたトピック名と完全一致するものは使用禁止

## チェック2: キーワードの重複
- `docs/topic-history.md` の Keywords 列を参照
- 新しいトピックのキーワードが、既存トピックのキーワードと **3つ以上一致** する場合はNG
- 例: 既出「chocolate, Maya, Aztec, cacao, European traders, sugar」
  - NG: 「cocoa, Aztec, Maya, beverages」→ 3つ一致（cocoa≒cacao, Aztec, Maya）
  - OK: 「tea, China, Tang dynasty, trade routes」→ 0つ一致

## チェック3: パッセージ内容の類似
- 同じタスクディレクトリ内の既存HTMLファイルを最低1つ読み、以下を確認:
  - 同じ歴史的事件・人物を扱っていないか
  - 同じ科学的概念・実験を説明していないか
  - 同じ構造（原因→結果→対策）で同系統の話題を論じていないか
- 一致する場合はトピックを変更して再生成

## チェック4: 学問分野の分散
- 直近3つの生成物と同じ学問分野にならないようにする
- 分野の例: 歴史、生物学、物理学、心理学、経済学、環境科学、社会学、芸術

## 記録
- 生成後、`docs/topic-history.md` にトピック名 + キーワード（5-6個）を必ず追記すること
